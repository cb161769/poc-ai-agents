Pitch ejecutivo

Un agente que escribe código no alcanza. Necesita poder rendir cuentas.

El problema

Lo que un agente autónomo normalmente no te puede responder

No es que los agentes de código fallen en silencio — es que, cuando funcionan, tampoco dejan nada para auditar después.

Pregunta 1

¿Qué información realmente vio el modelo?

Sin un punto de control de entrada, un ticket con un secreto pegado por error llega intacto al prompt — y nadie lo sabe hasta que ya es tarde.

Pregunta 2

¿Quién audita el resultado?

La mayoría de los pipelines confían en que el mismo agente que escribió el cambio también evalúe si está bien. No hay segunda opinión independiente.

Pregunta 3

¿Se acuerda de la última vez que esto salió mal?

Cada corrida empieza de cero. Un componente que ya filtró un secreto hace dos sprints no deja ninguna señal para la próxima vez que un agente lo toque.

La respuesta

Una cadena de evidencia, no una caja negra

Cada etapa deja un rastro real — un veredicto, un log, o un bloqueo — antes de que la siguiente pueda arrancar.

01
Ticket Jira real
Lee el ticket real — descripción, componentes, adjuntos — nunca un texto simulado.
02
AI Firewall gate
Redacta secretos reales y detecta intentos de jailbreak antes de que lleguen a cualquier modelo. Si rechaza, la corrida termina ahí — el agente ni se entera de que existía el ticket.
03
Contexto real
Grafo de dependencias en Neo4j, hallazgos vigentes de SonarQube, historial real de riesgo del componente — no un resumen inventado.
04
Coding agent + autocrítica
Cada escritura de archivo o comando pide confirmación humana en vivo. Antes de terminar, se autoevalúa — esa autocrítica viaja al juez, que la contrasta contra el diff real en vez de creerla.
05
Guardia de salida + tests reales block
Las mismas reglas del firewall, ahora aplicadas al diff que sale. El test suite real del proyecto corre en un contenedor descartable. Cualquiera de los dos puede bloquear antes de que el juez se moleste en mirar.
06
Juez independiente verifica
Modelo distinto, con acceso real a las mismas herramientas para comprobar afirmaciones en vez de confiar en el texto. Nunca vio el razonamiento del coding agent. Puede bloquear una corrida ya aprobada.
07
Veredicto ejecutado
OK → push real + PR real (abierta vía la REST API de Azure DevOps, no "pusheá a mano") + ticket a revisión. FLAGGED → bloqueo, comentario, rama marcada — nunca se mergea sola. Toda la corrida queda escrita como evidencia en el grafo de conocimiento.
08
Ciclo cerrado con revisión humana real
La PR abierta no es el final: un webhook entrante escucha los comentarios de revisión reales que un humano deja ahí, se los pasa al coding agent como feedback en la MISMA rama, y marca el thread como resuelto cuando aplica la corrección — sin que nadie tenga que volver a lanzar nada a mano.
Por qué esto, y no un framework de agentes

Tres decisiones que no se tomaron por default

Cada una se evaluó explícitamente contra la alternativa obvia — y se descartó con un motivo concreto, no por inercia.

01

Reversible por diseño

Nunca actúa sobre la rama principal. Cada escritura y cada comando de shell piden confirmación humana antes de ejecutarse — visible y respondible en vivo, no una casilla que se tildó una vez.

02

Adversarial a propósito

El juez y el coding agent son deliberadamente independientes: el juez nunca ve el razonamiento del agente, y nunca puede ser convencido después de decidir. No es una limitación técnica — es la garantía real.

03

Memoria institucional real

Cada corrida se escribe como evidencia en un grafo de conocimiento en Neo4j. El próximo agente que toque el mismo componente puede consultar su historial real de riesgo antes de actuar — no arranca de cero cada vez.

Qué hace falta para pasar de PoC a piloto

No pedimos adoptar un framework nuevo. Pedimos un repo real y una decisión de presupuesto.

Repo objetivo
Un repositorio real (no sample-repo/) donde el coding agent pueda operar sobre una rama nueva.
Proyecto Jira
Permisos de creación de issues confirmados, no solo lectura — el pipeline ya distingue entre ambos.
Presupuesto de modelo
Anthropic (mejor calidad de juicio) u Ollama local (gratis, offline) — el fallback en vivo ya está probado entre ambos.
